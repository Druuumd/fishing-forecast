# DOR / DOD для задач MVP

## Definition of Ready (DOR)
Задача берется в работу, если:
- есть понятная бизнес-цель;
- определены входы/выходы и зависимости;
- согласованы критерии приемки;
- есть оценка трудозатрат и рисков;
- обозначен владелец.

## Definition of Done (DOD)
Задача считается завершенной, если:
- выполнены acceptance criteria;
- есть тестовое покрытие требуемого уровня (unit/integration/smoke);
- обновлена документация по изменению;
- добавлены логи/метрики для операционной наблюдаемости;
- пройден code review;
- изменение развернуто в stage и проверено smoke-тестом.
