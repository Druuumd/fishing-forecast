# Quality Gates для Fishing Forecast MVP

## 1) Метрики модели (обязательные)

### Базовые метрики точности
- `MAE` и `RMSE` по каждому таргету (`pike`, `perch`) и в aggregate.
- Сравнение только на одинаковом срезе данных и одинаковом горизонте прогноза.

### Ранговые метрики (продуктовая ценность)
- `Spearman` по скору "удачности дня" >= 0.35 на валидации.
- `Top-day hit-rate@20%` >= 0.60:
  - среди дней, предсказанных как верхние 20%, не менее 60% должны попадать в фактические верхние 20%.

### Guardrails модели
- Drift по ключевым признакам не выше порога PSI 0.2 для автоматического retrain.
- При ухудшении любой основной метрики > 10% относительно текущего production baseline модель не продвигается в prod.

## 2) DQ-gates для данных
- `freshness`: все критичные источники обновлены не старше 24 часов.
- `completeness`: пропуски по обязательным полям < 1%.
- `range checks`: 0 критических нарушений диапазонов.
- `duplicates`: доля дублей < 0.5%.
- Любой красный DQ-статус блокирует релиз модели и API, использующих новый датасет.

## 3) Release Gates по фазам

### Gate A (Foundation)
- Утверждены ADR.
- Определены DOR/DOD.
- Зафиксированы риски и владельцы mitigations.

### Gate 1 (Infra)
- После reboot `docker compose` поднимает сервисы без ручного вмешательства.
- Health/readiness успешны 100% в 24-часовом окне.
- Security baseline закрыт.

### Gate 2 (Data Platform)
- 14 дней ingest без критических сбоев.
- DQ-отчет зеленый ежедневно.
- Есть fallback/override для температуры воды.

### Gate 3 (ML + API)
- API-контракт стабилен и покрыт smoke/integration.
- Модель проходит метрики качества и guardrails.
- `POST /catch` защищен rate limit + anti-spam.

### Gate 4 (Mobile)
- Пройдены сценарии offline/poor network/first-run/auth-fail.
- Crash-free sessions >= 99.0% на beta.
- p95 времени открытия экрана прогноза <= 2 сек на целевом устройстве.

### Gate 5 (Go-Live)
- Stage полностью зеленый перед продом.
- Проведен rollback drill.
- Подтверждены on-call и runbook.

## 4) Политика блокировки релиза
- Любой провал обязательного gate переводит релиз в `blocked`.
- Разблокировка только после документированного фикса и повторного прогона проверок.
- Временные исключения (waiver) допустимы только для некритичных критериев и с указанием срока погашения долга.
