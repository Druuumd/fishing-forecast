# ADR 0001: Foundation Decisions for MVP

## Статус
Accepted

## Контекст
Нужны базовые архитектурные решения для ускоренного вывода MVP с минимальными рисками в эксплуатации.

## Решения

### 1) Auth
- Для MVP использовать email/password + JWT access token.
- Refresh token хранить безопасно, с возможностью отзыва.

### 2) Cache
- Использовать Redis для кэширования `GET /forecast`.
- TTL базового прогноза: 1 час.
- При недоступности Redis API продолжает работать без кэша (degraded mode).

### 3) Retrain policy
- Автоматический retrain разрешается только при:
  - достаточном объеме новых данных;
  - отсутствии критичного drift;
  - прохождении quality gates.
- Иначе используется текущая production-модель.

### 4) Backup policy
- Ежедневный backup БД, ежемесячный restore drill.
- Хранение нескольких поколений backup согласно ops-политике.

## Последствия
- Уменьшаем риск нестабильного релиза ценой умеренного роста операционной дисциплины.
- Закладываем базу для scale-out без усложнения MVP.
