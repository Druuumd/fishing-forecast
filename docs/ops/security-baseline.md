# Security Baseline (Phase 1)

Минимальные требования до перехода к Фазе 2.

## 1) Хост и сеть
- UFW включен, разрешены только нужные порты (`22`, `80`, `443`, `8000` для stage при необходимости).
- `fail2ban` установлен и активен для SSH.
- SSH доступ:
  - отключен парольный логин;
  - разрешен только key-based auth.

## 2) Системная безопасность
- Автообновления security patches включены.
- Отдельный non-root пользователь для деплоя.
- Логи auth и system доступны для аудита.

## 3) Контейнеры
- Приложение запускается под non-root user (реализовано в `backend/Dockerfile`).
- Нет секретов в репозитории и в образе.
- Healthcheck активен в Compose.

## 4) API-безопасность (MVP минимум)
- Базовый rate-limit на чувствительных endpoint (в первую очередь `POST /catch`) - к реализации в Фазе 3.
- Валидация входных данных и ограничение payload.
- Trace-id в логах для расследования инцидентов.

## 5) Критерии прохождения baseline
- Чек-лист разделов 1-4 закрыт.
- Есть доказательства (команды/скриншоты/логи) в release notes.
- Назначен ответственный за поддержку baseline.
