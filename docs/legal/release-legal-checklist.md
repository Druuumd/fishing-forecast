# Legal & Privacy: минимальный пакет к релизу MVP

## 1) Обязательные документы
- `[x]` `Privacy Policy`:
  - какие данные собираются (аккаунт, геолокация, уловы, технические логи);
  - цели обработки;
  - сроки хранения;
  - основания обработки;
  - контакты для запросов по данным.
- `[x]` `Terms of Service`:
  - правила использования сервиса;
  - ограничения ответственности;
  - допустимое поведение пользователя;
  - порядок изменений условий.
- `[x]` `Cookie/Tracking notice` (для API-first MVP, web-tracking не включен):
  - какие трекеры используются;
  - как отключить/ограничить.
- `[x]` Публичные legal контакты/URL:
  - backend endpoint `GET /v1/legal/info` готов и возвращает production значения для `kvh-forecast.ru`.

## 2) Consent и пользовательские права
- `[~]` Явное согласие на обработку геоданных в приложении (backend API готов, UI-flow в mobile pending).
- `[~]` Отдельное согласие на push-уведомления (backend API готов, UI-flow в mobile pending).
- `[~]` Понятный flow удаления аккаунта и пользовательских данных (backend API self-service готов, UI-flow в mobile pending).
- `[~]` Канал для DSAR-запросов (контакты + backend self-service API готовы, mobile UX pending).

## 3) Data governance минимум
- `[x]` Data inventory:
  - где хранится каждый тип данных;
  - кто имеет доступ;
  - срок хранения/удаления.
- `[~]` Процедура удаления данных:
  - soft delete для UX;
  - гарантированное hard delete по запросу в регламентный срок.
- `[x]` Логи доступа к админ-действиям и операциям удаления.

## 4) Security/Privacy controls
- `[~]` Секреты не хранятся в репозитории (требуется финальный аудит перед публичным релизом).
- `[ ]` Ротация ключей и паролей по регламенту.
- `[ ]` TLS для внешнего трафика.
- `[~]` Минимально необходимый доступ по ролям (частично, MVP).
- `[x]` Ограничение rate limit + abuse prevention для чувствительных endpoint.

## 5) Gate для релиза (Legal Go/No-Go)
Релиз блокируется, если не выполнено хотя бы одно:
- опубликованы Privacy Policy и Terms;
- реализовано и протестировано удаление аккаунта/данных;
- согласие на геоданные и push отображается корректно;
- назначен ответственный за legal/privacy инциденты;
- подтверждена процедура обработки пользовательских запросов по данным.
