# Data Deletion Policy (MVP)

Дата обновления: 2026-04-21

## 1) Что можно удалить
- Пользовательские записи уловов.
- Связанные пользовательские идентификаторы в прикладных данных.
- Экспорт персональных данных пользователя доступен через:
  - `GET /v1/me/data`.

## 2) Канал запроса удаления
- Email публикуется через `GET /v1/legal/info` (`contact_email`).
- В запросе необходимо указать идентификатор пользователя и подтверждение владения.
- Для авторизованного пользователя в MVP доступен API self-service:
  - `DELETE /v1/me/data` (удаление уловов и consent-профиля).

## 3) SLA обработки
- Срок обработки запроса на удаление: до 30 календарных дней.

## 4) Процедура
1. Получение и верификация запроса.
2. Удаление прикладных пользовательских данных из БД.
3. Подтверждение пользователю о завершении.
4. Технические логи и backup очищаются/естественно истекают согласно retention-политике.

## 5) Ограничения
- Часть технических событий может временно сохраняться в логах до окончания retention-срока.
- Данные в backup могут существовать до истечения окна хранения (30 дней в MVP).
